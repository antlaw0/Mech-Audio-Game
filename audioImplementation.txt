# ✅ **RANGED_COMBAT_AUDIO_UPDATE.md**

### **Audio-First Ranged Combat System (Fair, Trackable, Reactable)**

## Purpose

Convert ranged combat into a **fully trackable audio timeline** so the player can:

* detect threats early
* understand direction and distance
* react in time
* dodge or counter

All ranged attacks must follow a **multi-stage audio pipeline**.

---

# 1. GLOBAL CONSTANTS

Create in `combatConfig.ts`:

```ts
export const MIN_REACTION_TIME = 0.4 // seconds
export const IDEAL_REACTION_TIME_MIN = 0.6
export const IDEAL_REACTION_TIME_MAX = 0.9

export const PROJECTILE_SPEED = 6.0 // tiles per second
export const PROJECTILE_RADIUS = 0.2 // tiles
export const PLAYER_RADIUS = 0.25 // tiles

export const NEAR_MISS_RADIUS = 1.2 // tiles
export const MAX_PROJECTILE_RANGE = 20 // tiles

export const ENEMY_FIRE_COOLDOWN = 2.0 // seconds
```

---

# 2. ATTACK STATE MACHINE

Each enemy must implement:

```ts
type AttackState =
  | "idle"
  | "intent"
  | "lock"
  | "fire"
  | "cooldown"
```

Add to enemy class:

```ts
attackState: AttackState
attackTimer: number
```

---

# 3. FULL ATTACK TIMELINE

## 3.1 Intent Phase (Pre-Telegraph)

Duration: **0.3 – 0.6 seconds**

### Behavior:

* Start when enemy decides to attack
* Play **charging sound**
* Sound must be positional

### Audio rules:

* Pitch rises over time
* Volume stable
* Pan based on enemy position

```ts
pitch = basePitch + (progress * 200)
```

---

## 3.2 Lock Phase (Target Confirmation)

Duration: **0.15 – 0.3 seconds**

### Behavior:

* Enemy has line-of-sight to player
* Emits repeating “lock tick”

### Audio:

* Short chirp every 0.25s
* Directional
* Slightly brighter than idle sounds

---

## 3.3 Fire Event

Instant transition

### Behavior:

* Spawn projectile
* Play loud firing sound

### Audio:

* Strong transient (fast attack)
* Clear stereo positioning

---

## 3.4 Projectile Travel (MOST IMPORTANT)

Every projectile must emit continuous sound.

### Create projectile object:

```ts
{
  x: number,
  y: number,
  vx: number,
  vy: number,
  active: boolean,
  audioSourceId: string
}
```

---

### Movement:

```ts
x += vx * delta
y += vy * delta
```

---

### Audio behavior (every frame):

#### Distance-based volume:

```ts
const distance = getDistance(projectile, player)
const volume = clamp(1 - (distance / MAX_PROJECTILE_RANGE), 0.05, 1)
```

#### Pitch based on speed:

```ts
const speed = Math.sqrt(vx * vx + vy * vy)
const pitch = 200 + (speed * 50)
```

#### Stereo panning:

```ts
const bearing = getBearing(player.angle, projectile)
panner.pan.value = Math.sin(bearing)
```

---

# 4. NEAR MISS SYSTEM

If projectile passes close to player:

```ts
if (distanceToPlayer < NEAR_MISS_RADIUS && !projectile.hasPlayedNearMiss)
```

### Then:

* Play fast “whoosh past” sound
* Pan strongly to passing side
* Mark `hasPlayedNearMiss = true`

---

# 5. HIT DETECTION

Collision when:

```ts
distance(projectile, player) < (PLAYER_RADIUS + PROJECTILE_RADIUS)
```

### On hit:

* Play impact sound (non-positional or centered)
* Apply damage
* Destroy projectile

---

# 6. MISS CLEANUP

If projectile exceeds:

```ts
distanceFromOrigin > MAX_PROJECTILE_RANGE
```

Then:

* Stop audio
* Remove projectile

---

# 7. ENEMY AIMING (FAIRNESS SYSTEM)

Enemies must NOT have perfect accuracy.

### Add spread:

```ts
const spreadAngle = randomRange(-0.1, 0.1) // radians (~6 degrees)
```

Apply to firing angle.

---

### Optional difficulty scaling:

```ts
easy:   ±0.15
normal: ±0.1
hard:   ±0.05
```

---

# 8. FIRE RATE CONTROL

Enemy cannot fire continuously.

```ts
if (timeSinceLastShot < ENEMY_FIRE_COOLDOWN) return
```

---

# 9. LINE OF SIGHT CHECK

Enemy may only enter **lock phase** if:

```ts
hasLineOfSight(enemy, player) === true
```

Reuse your raycasting system.

---

# 10. AUDIO PRIORITY RULES

Projectile audio must:

* override ambient noise
* be louder than obstacle tones within 8 tiles
* never be culled by obstacle density filter

---

# 11. REQUIRED AUDIO FILES OR SYNTHS

If no files exist, use Tone.js:

```ts
Intent:        rising sine synth
Lock:          short square blip
Fire:          noise burst + sine click
Projectile:    continuous filtered sawtooth
Near miss:     fast filtered noise sweep
Impact:        low thud (membrane synth)
```

---

# 12. REQUIRED FUNCTIONS

Ensure these exist:

```ts
spawnProjectile(enemy, player): Projectile
updateProjectile(projectile, delta): void
playIntentSound(enemy): void
playLockTick(enemy): void
playFireSound(enemy): void
playProjectileLoop(projectile): void
playNearMiss(projectile): void
playImpact(): void
```

---

# 13. INTEGRATION

Copilot must:

1. Add attack state machine to all ranged enemies
2. Add projectile system to game loop
3. Update audio every frame
4. Ensure positional audio is correct
5. Ensure timing meets reaction window
6. Prevent instant/untelegraphed damage

---

# 14. SUCCESS CRITERIA

The system is correct if:

* Player can hear attack BEFORE it happens
* Player can track projectile in space
* Player can dodge based on sound
* Hits never feel random
* Combat feels fair and readable
