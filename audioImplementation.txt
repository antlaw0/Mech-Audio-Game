# 2. RADAR–SONAR SYSTEM (PRIMARY AWARENESS)

You must implement **two layers**, like Swamp:

## Layer A — Passive Sweeping Sonar (continuous)

A slow 360° rotating beam.

### Behavior

* The sweep turns at a fixed speed = **one full rotation every 2.5 seconds**
* It casts **one ray per sonar tick** (NOT per frame)
* Only objects up to **12 tiles away** may respond
* Sweep direction = clockwise

### Audio cue from the sweep beam

* A soft oscillating tone (already implemented)
* Pitch rises slightly as the sweep angle increases
* Must be low volume

---

## Layer B — Active Sonar Ping (manual ping like Swamp)

Triggered by user input.

### Behavior

* Emits **16 rays** in a starburst at equal angular spacing
* Each ray reports **the nearest obstacle OR nearest enemy**, whichever is closer
* Obstacles respond with short filtered blips
* Enemies respond with stronger signature pings

### Active ping ranges

```
Obstacle detection: 12 tiles max
Enemy detection: 20 tiles max
```

# 3. OBSTACLE TONAL RESPONSES (SWAMP-STYLE)

Obstacles responding to sonar must produce **tonal pips** based on:

### 3.1 Distance → Volume

Use this falloff formula:

```
volume = clamp(1 - (distance / OBSTACLE_AUDIO_MAX_DISTANCE), 0.05, 1)
```

### 3.2 Type of obstacle → Timbre

Copilot must give each 1-tile cell type its own tone:

```
Wall (cell=1): sine tone at 480Hz
Cover block: triangle at 520Hz
Tall rock: square at 420Hz
Boundary wall: sawtooth at 400Hz
```

(Use placeholder synthesis unless real assets exist.)

### 3.3 Bearing → Stereo position

Determine stereo pan using:

```
bearing = angleBetween(playerFacing, directionToObstacle)
panner.pan.value = sin(bearing)
```

### 3.4 Elevation (if any)

If elevation is used later:
higher = slightly brighter (increase filter cutoff).

# 4. OBSTACLE DENSITY FILTER (THE BIG MISSING PIECE)

Swamp solves “too many obstacle sounds” using a density-limited nearest-neighbor filter.

You must implement the same rule:

### Rule:

**At any sonar tick, only the closest N obstacles may make sound.**

```
MAX_SIMULTANEOUS_OBSTACLE_TONES = 5
```

Steps:

1. Collect all obstacles hit by sonar beams
2. Sort by distance ascending
3. Keep top 5
4. Discard any further sounds entirely
5. For the survivors, play their tones normally

This is critical—without it the audio becomes cluttered.

---

# 5. NEAR-FIELD COLLISION AWARENESS (LIKE SWAMP)

When the player mech is within **1.5 tiles** of a wall:

### Always play a soft “wall proximity buzz”

* filtered at 300Hz
* volume based on inverse distance
* stronger on the side the wall is on

### If the player **touches** the wall (collides):

* play a short muted thud
* pan toward the impact side

Copilot must call this **every frame**, not per sonar tick.

# 6. CARDINAL DIRECTION ORIENTATION CUES

Your map uses tiles, so cardinal alignment is important.

Implement a Swamp-style system:

### Pressing the orientation keys snaps facing to:

```
0°   = North (toward row decreasing)
90°  = East  (col increasing)
180° = South (row increasing)
270° = West  (col decreasing)
```

### Each snap produces a distinct audio cue:

* North: rising triad (C–E–G)
* East: single bright ping
* South: descending triad
* West: soft double-click

Copilot must implement a small utility to produce these.

# 7. ENEMY AWARENESS (TANK, MECH, ETC.)

Enemies use a **different audio channel** than obstacles.

## 7.1 Enemy Idle Presence

Always on if within 25 tiles.

Tank example:

* looping treads
* low-pass filter when around corners

## 7.2 Enemy Line-of-sight Check

If enemy has LOS to player:

* play a brief “lock-on tick”
* every 300ms
* must be directional

## 7.3 Enemy Movement Sound

Pitch reflects speed:

```
pitch = basePitch + (velocity * 12)
```

### Bearing pan:

Same rule as obstacles.

# 8. SONAR → ENEMY RESPONSE PRIORITY

When the sweep beam hits both an obstacle and enemy in the same direction:

**Enemy wins**.

Rule:

```
if enemyDistance < obstacleDistance + 1 tile:
    enemy responds
else:
    obstacle responds
```

This mirrors Swamp’s “enemies override clutter” behavior.
