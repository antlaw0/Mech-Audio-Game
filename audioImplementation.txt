
# ✔️ Instructions for Copilot (paste directly)

**Task:**
Implement a sweeping audio-sonar system similar to the sweeping sonar in the audio game *Swamp*. Use **Tone.js** for audio. The sonar must simulate a rotating “beam” sweeping left to right across the player’s forward view and use raycasting results to shape the audio. The audio must convey walls, corridors, open space, and corners through volume, filtering, and panning.

**Build everything inside a single JavaScript module** with clean, readable functions.
I want accurate, working code.
All blocks must have correct matching braces, and every closing brace must include a trailing comment explaining what it closes.

---

## 1. Architecture requirements

Implement the following:

### A. Player structure

```javascript
player = {
    x: number,
    y: number,
    facing: number // in degrees
}
```

### B. Raycasting system

Implement a simple 2D raycast function:

`raycast(x, y, angleDeg): number`

Rules:

* Converts angle to radians.
* Steps forward in small increments (0.1 to 0.2 units).
* Stops when hitting a wall tile in a 2D grid map.
* Returns distance to the wall.
* Must not hang if no wall is hit; clamp to max distance.

### C. Sweeping sonar controller

Create a `SweepingSonar` class with:

* `start()`
* `stop()`
* `update()` (should run every 20 ms via requestAnimationFrame or setInterval)
* Internally maintains:

  * sweepAngle (current relative angle, starting at -45 degrees)
  * sweepDirection (1 for rightward sweep, -1 for leftward sweep)
  * sweepSpeed (degrees per update)
* Sweep oscillates between -45 and +45 degrees.

### D. Audio engine (Tone.js)

Create the following:

* One continuous oscillator at about 1200 Hz.
* A lowpass filter to represent distance.
* A stereo panner.
* Audio mappings:

  * Closer walls = louder + brighter filter cutoff.
  * Farther walls = quieter + darker filter cutoff.
* Panning corresponds to sweep angle:

  * -45 degrees pan = -1
  * +45 degrees pan = +1

Audio must update in real time with each sweep angle step.

### E. Controls

Provide simple methods the game can call:

* `setEnabled(true/false)` to turn the sweeping sonar on or off.
* `updatePlayerPosition(x, y, facingDeg)`.

## 2. Sweeping logic details

### World angle calculation:

```
worldAngle = player.facing + sweepAngle
```

### Raycasting distance mapping:

Given `distance` and a `maxDistance` value (e.g., 30):

Compute:

```
closeness = 1 - clamp(distance / maxDistance, 0, 1)
```

Then:

```
volume = -30 + (closeness * 20)  // in decibels
cutoff = 1000 + (closeness * 8000)
pan = sweepAngle / 45
```

Apply these values to the Tone.js nodes on every update.
